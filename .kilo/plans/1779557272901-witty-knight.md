# Plan: DRY JSON-LD Refactoring — astrowind-imprezowy

## Problem

Every page in `src/pages/oferta/` contains **inline** `<script type="application/ld+json">` blocks with:
- Duplicated `areaServed` arrays (6 cities + Polska) repeated on **every** page
- Duplicated `priceRange`, `@context` declarations
- Inline FAQ schemas that duplicate FAQ content already in the page
- No shared `LocalBusiness` entity — each page declares its own isolated `ProfessionalService`
- No `@graph` structure, no `@id` cross-referencing between entities
- No centralized business data source (address, phone, geo, sameAs, etc.)

## Reference Solution (pineaddle-com)

The reference project uses a **centralized schema factory** pattern:
1. `src/data/business.ts` — single source of truth for all business data
2. `src/utils/schema.ts` — factory functions (`createWebSiteSchema`, `createLocalBusinessSchema`, `createServiceSchema`, `createFAQSchema`, `createBreadcrumbSchema`, `createGraph`, `createOfferPageGraph`, etc.)
3. Each page calls a graph builder (e.g. `createOfferPageGraph(config)`) and renders the result as a single `<script type="application/ld+json">`

### Step 2: Create `src/utils/schema.ts`

Copy the reference project's `schema.ts` and adapt it for this project's domain, URL structure, and service catalog. Key functions:

| Function | Purpose |
|---|---|
| `createWebSiteSchema()` | WebSite entity with `@id` |
| `createWebPageSchema(config)` | WebPage with breadcrumb, about, mainEntity, dates |
| `createLocalBusinessSchema(overrides?)` | LocalBusiness/ProfessionalService with all fields as `@id` references |
| `createServiceSchema(config)` | Service schema for individual offer pages |
| `createFAQSchema(url, questions)` | FAQPage schema |
| `createBreadcrumbSchema(url, items)` | BreadcrumbList schema |
| `createGraph(...schemas)` | Wraps schemas in `@context` + `@graph` |
| `createAddressSchema()` | PostalAddress entity |
| `createGeoSchema()` | GeoCoordinates entity |
| `createOpeningHoursSchema()` | OpeningHoursSpecification entity |
| `createImageSchemas()` | ImageObject entities for logo + primary image |
| `createOfferCatalogSchema()` | OfferCatalog with all 12 services from oferta page |
| `createOfferPageGraph(config)` | **Main factory** — builds complete @graph for any offer page |
| `createHomePageSchema()` | Complete @graph for the homepage/oferta index |

## Key Differences from Reference Project

1. **No reviews/rating** — this project doesn't have aggregateRating or reviews in its current JSON-LD, so those entities will be omitted (or added later if needed)
2. **No founders/contactPoints** — simpler business structure, no need for ContactPoint or founder entities
3. **Different domain** — `imprezowyfotograf.pl` vs `pineaddle.com`
4. **Different service catalog** — 12 event photography services vs 4 studio services
5. **Simplified `createOfferPageGraph`** — won't include all the reference project's entities (founders, reviews, etc.) unless they're relevant
